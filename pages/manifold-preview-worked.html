Worked-example format brainstorm

Each step's worked example takes the previous step's output as input, substitutes real numbers into the formula, and shows this step's output. Two candidate formats below, demonstrated on two very different step types: PCA centering (point-wise) and Isomap double-centering (matrix-wise).

Example A · PCA step 1 (centering, point-wise)

Operation: subtract the sample mean from every point. Input is the raw cloud from step 0; output is the centered cloud.

Format 1 · Sectioned blocks

Input (from step 0)
3 of N=150 raw points: x_5 = (-7.231, 9.842, 11.504) x_27 = ( 4.118, 12.730, -2.336) x_92 = ( 1.605, 3.487, 8.219) mean of all N points: μ = (-0.412, 9.823, 1.077)
Formula
$$x_i \leftarrow x_i - \mu$$
Output (after centering)
x_5' = (-7.231 - (-0.412), 9.842 - 9.823, 11.504 - 1.077) = (-6.819, 0.019, 10.427) x_27' = ( 4.118 - (-0.412), 12.730 - 9.823, -2.336 - 1.077) = ( 4.530, 2.907, -3.413) x_92' = ( 1.605 - (-0.412), 3.487 - 9.823, 8.219 - 1.077) = ( 2.017, -6.336, 7.142)
Three clearly labelled blocks. Each row shows the actual subtraction and the result. The mean is computed once and reused.

Format 2 · Inline substitution

$$x_i \leftarrow x_i - \mu \quad\text{with}\quad \mu = (-0.412,\, 9.823,\, 1.077)$$
x_5 = (-7.231, 9.842, 11.504) − μ → (-6.819, 0.019, 10.427) x_27 = ( 4.118, 12.730, -2.336) − μ → ( 4.530, 2.907, -3.413) x_92 = ( 1.605, 3.487, 8.219) − μ → ( 2.017, -6.336, 7.142)
Inputs in light blue, output in warm yellow, with the operator inline between them.

Example B · Isomap step 4 (double-centering, matrix-wise)

Operation: form the Gram matrix B by double-centering the squared distance matrix D^2. Input is the geodesic distance matrix from step 3; output is the centered Gram B passed to step 5.

Format 1 · Sectioned blocks

Input (from step 3)
D^2 (squared geodesic distances, 4 of N=150 rows shown): [ 0.00, 1.18, 3.20, 4.91, ... ] [ 1.18, 0.00, 0.96, 2.83, ... ] [ 3.20, 0.96, 0.00, 1.44, ... ] [ 4.91, 2.83, 1.44, 0.00, ... ] row means r: [ 2.94, 2.41, 2.18, 2.83, ... ] column means c: [ 2.94, 2.41, 2.18, 2.83, ... ] grand mean g: 2.59
Formula
$$B_{ij} = -\tfrac{1}{2}\bigl(D^2_{ij} - r_i - c_j + g\bigr)$$
Output (Gram matrix B)
Example B[1][2] computation: = -1/2 * (D^2[1][2] - r[1] - c[2] + g) = -1/2 * ( 0.96 - 2.41 - 2.18 + 2.59) = -1/2 * (-1.04) = 0.520 B (4 of N=150 rows shown): [ 1.42, -0.06, -0.27, -1.09, ... ] [ -0.06, 0.74, 0.52, -0.21, ... ] [ -0.27, 0.52, 1.07, 0.38, ... ] [ -1.09, -0.21, 0.38, 1.46, ... ]

Format 2 · Inline substitution

$$B_{ij} = -\tfrac{1}{2}\bigl(D^2_{ij} - r_i - c_j + g\bigr)$$
means computed once: r = [2.94, 2.41, 2.18, 2.83, ...] c = [2.94, 2.41, 2.18, 2.83, ...] g = 2.59 B[1][2]: -1/2 (0.96 − 2.41 − 2.18 + 2.59) → 0.520 B[2][1]: -1/2 (0.96 − 2.18 − 2.41 + 2.59) → 0.520 B[3][0]: -1/2 (4.91 − 2.83 − 2.94 + 2.59) → -1.090 full B (4 of N=150 rows): [ 1.42, -0.06, -0.27, -1.09, ... ] [ -0.06, 0.74, 0.52, -0.21, ... ] [ -0.27, 0.52, 1.07, 0.38, ... ] [ -1.09, -0.21, 0.38, 1.46, ... ]
For matrix steps, the inline format still shows a couple of concrete element computations followed by a heatmap-style excerpt of the full output.

Trade-offs

  • Format 1 (sectioned blocks): clearest separation between Input, Formula, and Output. Reads top-to-bottom like a textbook. Takes more vertical space.
  • Format 2 (inline substitution): denser. Inputs and outputs are colour-coded so the eye can follow the substitution line by line. Best for point-wise operations; tolerable for matrix ops with a small expansion at the end.
  • Pick one format that we use uniformly across every step (so the worked example is predictable to read), or pick a hybrid (e.g., format 1 for matrix-heavy steps and format 2 for point-wise steps).
